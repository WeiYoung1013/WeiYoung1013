Building Generalist Robots with Embodied Foundation Models

A working note on where large models help robots, where they still fail, and what I would measure first.

The Useful Part

Foundation models become valuable for robotics when they compress messy observations into reusable structure: object identity, affordance, intent, temporal context, and the language humans use to specify goals.

For an embodied agent, the core question is not whether a model can describe the world. The question is whether its representation can improve action selection under latency, uncertainty, and contact-rich dynamics.

Where I Would Start

  • Use language as a task interface, not as a replacement for low-level control.
  • Keep perception outputs grounded in geometry, state estimation, and uncertainty.
  • Evaluate policies by recovery behavior, not only by average task success.
  • Track the failure mode: perception miss, planning error, action execution, or distribution shift.

Open Problems

The strongest systems will likely combine pretrained multimodal priors with robot-specific data. The main gap is still action grounding: mapping a general representation into stable motor behavior across embodiment, environment, and time.

My current bias: generalist robot systems need model breadth, but deployment quality still comes from careful interfaces, datasets, and evaluation loops.
